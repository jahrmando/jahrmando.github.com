Renombrar interfaz de red en CentOS

05 Apr 2013

Es común que al cambiar físicamente una interfaz de red el sistema nos configure este nuevo hardware asignándole una ID de interfaz eth(X), nos saldrá el error anterior ya que el dispositivo con ID eth0 que estaba configurada no existe, si nosotros queremos usar ese mismo ID de interfaz haga los siguiente.

Hacemos un respaldo a la configuración

$ cp /etc/udev/rules.d/70-persistent-net.rules 70-persistent-net.rules.backup

Es probable que udev nos haya agregado la linea (12) con el nuevo dispositivo y asignado una ID para la nueva interfaz que hemos colocado, en la opción “NAME” lo cambiamos por eth0 y comentamos o eliminamos la linea (9) donde estaba asignado “NAME” como eth0

$ vim /etc/udev/rules.d/70-persistent-net.rules
# This file was automatically generated by the /lib/udev/write_net_rules
# program, run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single
# line, and change only the value of the NAME= key.

# PCI device 0x8086:0x100e (e1000)
#SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="08:00:27:4b:4b:48", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"

# PCI device 0x8086:0x100e (e1000)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="08:00:27:53:c5:28", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"

Lo que sigue es asignar el numero de MAC que tenemos en el archivo anterior a la configuracion de red de la interfaz eth0.

$ vim /etc/sysconfig/network-scripts/ifcfg-eth0 
HWADDR="08:00:27:53:C5:28"

Reiniciamos el sistema para poder aplicar los cambios.
